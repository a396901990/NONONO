JVM虚拟机
===========

ClassLoader（类加载器）
-------------
1.编写java代码  
2.将代码编译成class文件  
3.将class文件装载到内存中  

Java虚拟机与程序的生命周期
在如下几种情况下，Java虚拟机将结束生命周期：
* 执行了System.exit()方法  
* 程序正常执行结束  
* 程序遇到异常或错误而终止  
* 操作系统出现错误而导致JVM进程终止  

类的加载、连接与初始化
-----------------
* 加载：查找并加载累的二进制数据  
* 连接：  
    * 验证：确保被加载的类的正确性  
    * 准确：为类的**静态变量**分配内存，并将其初始化默认值  
    * 解析：把类中的符号引用转换为直接引用  
* 初始化：为类的静态变量赋予正确的初始值

Java虚拟机必须在每个类或接口被Java程序**首次主动使用**时才初始化他们，如：  
    
    // 创建类的实例  
    new Test();

    // 访问某个类或接口的静态方法，或者对该静态变量赋值   
    int b = Test.a;
    Test.b = b; 

    // 调用类的静态方法
    Test.doSomething();

    // 反射
    Class.forName("com.dean.Test");
    
    // 初始化一个类的子类
    Class Parent{}
    Class Child extents Parent {
        public static int a = 3;
    }
    Child.a = 4;
    // Java虚拟机启动时被标明为启动类的类
